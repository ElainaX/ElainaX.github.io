颜色空间分为两种

  1. Linear — 线性颜色空间，是执行颜色间计算的主空间

# Game Object

游戏物体默认创建时具有以下几个属性

  • Transform — 控制物体的变换位置

# 脚本编码

# C# Script Asset

每一个创建用来控制游戏物体（Game Object）的类都需要继承 MonoBehaviour 父类（在 UnityEngine 命名空间中）

using UnityEngine;
public class Clock : MonoBehaviour {}

为了让这个类控制 Clock 的 Transform 变量行为，我们需要声明一个 Transform 成员来引用绑定 Clock 的 Transform 变量

[SerializeField]
Transform hoursPivot;

上述代码的 “[SerializeField]” 标识符表示这个这个变量可以被序列化，unity 中序列化会使得 inspector 可以获取到这个变量的信息并显示出来，注意：Unity 中共有字段（public）可以直接被 inspector 获取和显示，所以 “[SerializeField]” 主要修饰非公有字段

# Unity 中的类函数

MonoBehaviour：

这个父类可以让我们实现（implement）以下接口

  • Awake () — 指示游戏启动的时候游戏对象的行为
  • Update () — 指示每帧（Frame）游戏对象如何更新资源和变量

还有很多其他的可以实现的接口，但是我们本章不需要了解

Debug：

我们想要在控制台打印一些游戏的状态信息，这时候就需要引入 System 命名空间，这个命名空间中包含 Debug 类，Debug 类中有 Log 函数可以打印我们需要的东西

时间：

我们想要获取当前时间，System 中的 DateTime.Now 属性会返回一个 DateTime 类型，里面包含了当前的日期和时间

旋转角：

我们可以使用 Quaternion（四元数）里的 Euler 函数来定义欧拉角旋转，这会返回我们一个指向那个角度的方向向量（欧拉角也就是绕着三个轴分别旋转的度数，和我们的 Transform 匹配）

运用以上的类函数，我们就可以打印当前时间的同时使用欧拉角控制我们物体的旋转

using System;
using UnityEngine;

public class Clock : MonoBehaviour {

	[SerializeField]
	Transform hoursPivot, minutesPivot, secondsPivot;

	void Awake () {
		Debug.Log(DateTime.Now);
		hoursPivot.localRotation = Quaternion.Euler(0, 0, -30);
	}
}

以上代码值得注意的点就是 localRotation 而不是 rotation 属性，这是因为 rotation 指的是全局旋转角（包括父物体的旋转角），而 localRotation 属性则单指当前物体的旋转角。

获取连续时间：

DateTime 的 Now 中的 Hours、Minutes 和 Seconds 对象只能返回到秒（也就是整数）的改变，如果我们想要更精确的时间（小数），可以使用 Now 中的 TimeOfDay 变量，它是一个 TimeSpan 对象，可以得到时间的小数，就能让我们钟表移动的更平滑，但要注意 Unity 中所有的对象默认都是 float 运算，但是这里 TimeOfDay 返回的小数确是 double 类型的，所以为了成功运行我们的程序我们需要强制类型转换

hoursPivot.localRotation =
			Quaternion.Euler(0f, 0f, hoursToDegrees * (float)time.TotalHours);
		minutesPivot.localRotation =
			Quaternion.Euler(0f, 0f, minutesToDegrees * (float)time.TotalMinutes);
		secondsPivot.localRotation =
			Quaternion.Euler(0f, 0f, secondsToDegrees * (float)time.TotalSeconds);

# 生成实例（Instance）

# Prefabs

只需要在 Assets 界面创建游戏物体，我们就算是创建好了一个 prefabs， 我们需要用他来生成实例，要手动创建实例，只需要将该 prefabs 拖拽到视图中或者层次窗口中

# 使用脚本

使用空物体来创建实例，我们将 Graph 脚本（里面有一个 Transform 属性）绑定到空物体上，再把他的 Transform 属性链接给我们创建好的 Point Prefabs，这样代码中执行 Instantiate 命令的时候，就可以实例化我们绑定好的 Point Prefabs，因为每个 GameObject 的属性都会与它本身的引用相绑定，所以实例化属性也就相当于实例化了一个 GameObject，只不过我们现在只使用它的 Transform 属性

public class Graph : MonoBehaviour {

	[SerializeField]
	Transform pointPrefab;
	
	void Awake () {
		Transform point = Instantiate(pointPrefab);
        point.localPosition = Vector3.right;

		point = Instantiate(pointPrefab);
		point.localPosition = Vector3.right * 2f;
	}
}

上面代码中的 Vector3.right 会返回一个 (1, 0, 0) 的向量，因为 unity 使用左手系，x 轴指向右边，y 轴指向上边，z 轴顺着视线方向向屏幕里指去。

# 范围条

在 Unity 中可以使用 Range 标识符来表示生成一个可以拖动的范围条

[SerializeField, Range(10, 100)]
int resolution = 10;

# 设置父类

生成的 Point 实例目前以他们自己为根物体，但是我们希望通过将他们的父类设置为生成他们的 GameObject 来更好的控制整体变换，我们可以使用 SetParent 方法来实现

for (int i = 0; i < resolution; i++) {
			…
			point.SetParent(transform, false);
		}

因为我们把当前脚本（Graph.cs）绑定给了 Graph 这个 GameObject，所以直接引用 transform 就是引用的当前 GameObject 的 transform，第二个 false 代表我们不需要调整物体的位置（默认情况下 unity 会改变 local 的 Transform 属性以匹配到物体原始 Global 空间位置）

# Surface Shader

我们希望让这些点根据自己所在的位置变化颜色，也就是通过改变他们材质的 shader 来实现这点

创建 surface shader 可以通过 Assets / Create / Shader / Standard Surface Shader ，当前我们希望为 point 创建好一个 shader，所以我们叫他 Point Surface。

shader 文件的格式：

unity 使用 shaderLab 作为语言编写 shader，它是 HLSL 和 CG 语言的结合体，主要是 CG。

基本结构如下：

这里 Graph/Point Surface 是 shader 的名称，后面可以在 Material 中指定，整个 Shader 由多个 SubShader 组成，每个 SubShader 负责处理某个格式的表面着色的内容，unity 会逐一匹配他们，如果全部匹配失败，可以指定此时回调使用默认的 unity 着色器，比如 diffuse，这里 ==FallBack “Diffuse”== 的作用就是如此。

在 SubShader 中，所有的 shader 代码被 CGPROGRAM 和 ENDCG 包裹，shader 代码可以使用 #pragma 标识来配置相应的编译属性

#pragma surface ConfigureSurface Standard fullforwardshadow

这一句代码表示接下来的命令会配置表面相关属性和渲染参数（通过 surface 指明）

（ConfigureSurface）— 指明我们接下来要配置表面属性而不是渲染参数

（Standard）— 表示使用标准的表面着色模型，具有常用表面属性，这是 ==（ConfigureSurface）== 的参数

（fullforwardshadows）— 表示启用完整的前向渲染阴影，不会在延迟渲染路径中计算，这也是 ==（ConfigureSurface）== 的参数

#pragma target 3.0

这一句代码表示我们会使用 3.0 版本的 shaderLab 渲染

struct Input 
{
	float3 worldPos;	
};

float _Smoothness;

void ConfigureSurface(Input input, inout SurfaceOutputStandard surface) 
{ 
	surface.Albedo.rg = input.worldPos.xy * 0.5 + 0.5;
	surface.Smoothness = _Smoothness;
}

这段代码首先定义了一个 Input 结构体，里面有一个叫做 worldPos 的变量，我们将他传递到 ConfigureSurface 函数中，这个函数是 unity 用来配置表面属性的函数，我们将 Input 结构体当作他的参数，Unity 就会根据某些命名规则传递顶点信息给其中的变量，比如 worldPos 变量包含 pos 这个字段，那么 Unity 就会把顶点位置信息传递给 worldPos 这个变量

第二个参数 SurfaceOutputStandard 是 unity 一个内置的结构体，它主要包含表面的基础信息，inout 表示它会在这个函数中使用并且输出到下一个流水线状态
